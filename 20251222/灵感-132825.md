## ⚠️ 遇到的问题
1. 在vscode 中配置项目级别的keybinding.json 文件出现错误
错误信息：
```shell
Document c:\Users\P30015874206\Desktop\note\.vscode\keybindings.json not found in AST tracker
```
错误原因：
    1.配置文件没有加载成功
    2.配置文件格式错误或者书写存在错误

2.vscode中自动补全存在限制
    1.在项目级别的keybinding.json 文件中配置的快捷键不会自动补全
    2.在全局的keybinding.json 文件中配置的快捷键会自动补全

3.vscode中gui中的配置只是子集，还有一部分必须通过settings.json 文件进行配置
    1.在 VS Code 插件生态中，*package.json* 中的 contributes.configuration 字段是查询插件支持哪些配置项（settings）的最权威、最及时、最可靠的来源，其优先级甚至高于官方文档。
    2.code --list-extensions --show-versions 命令可以查看当前安装的插件以及其版本号,然后找到对应插件名使用listary查看插件的配置文件
    3.询问ai时有关闭自动补全的配置项,"enableInlineSuggestion": false,实际上通义并不支持
4.vscode中的插件是事件机制，而非轮询
    
5.在笔记项目中设置daynote 目录为git 子模块,可以单独进行管理
    1.可以实现手机上记录灵感笔记,并在pc上进行同步整合
